# 질문 통계 메뉴 백엔드 구축 계획

현재 `/stats` 페이지는 전부 모의 데이터입니다. 또한 학생이 보낸 질문이 어디에도 기록되지 않습니다. 이 계획은 **질문을 DB에 기록**하고, **규칙 기반 키워드 매칭으로 통계를 집계**해 통계 페이지를 실제 데이터로 채웁니다.

## 1. 데이터베이스: 질문 로그 테이블

`question_logs` 테이블을 새로 만듭니다.

- `question` — 학생이 입력한 질문 원문
- `answered` — 규칙에서 답을 찾았는지 여부 (정상 답변 / 못 찾음 구분용)
- (기본 필드: id, created_at)

접근 권한: 질문 기록·집계는 모두 서버 내부(관리자 권한)에서만 이뤄지므로, 외부 직접 접근은 막고 RLS를 켜되 일반 사용자용 정책은 두지 않습니다. (서버 함수만 데이터에 접근)

## 2. 질문 자동 기록

`askQuestion` 서버 함수(`src/lib/rag.functions.ts`)가 답변을 만들 때마다, 질문 원문과 답변 성공 여부를 `question_logs`에 저장합니다. 답변 생성을 막지 않도록 기록 실패는 무시(best-effort)합니다.

## 3. 규칙 기반 키워드 사전

`src/lib/stats-keywords.ts`에 키워드 사전을 정의합니다. 각 대표 키워드마다 동의어/유사 표현을 함께 둬서 매칭 정확도를 높입니다.

```text
휴대폰  ← 핸드폰, 폰, 스마트폰
실내화  ← 실내화, 슬리퍼
급식    ← 급식, 점심, 밥
지각    ← 지각, 늦, 등교시간
체육    ← 체육, 운동장, 강당
쉬는시간 ← 쉬는시간, 쉬는 시간, 휴식
청소    ← 청소, 당번
간식    ← 간식, 군것질
복도통행 ← 복도, 통행
화장실  ← 화장실, 양치
```

(교육 연구용으로 키워드는 이후 코드에서 쉽게 추가·수정 가능하도록 한 곳에 모읍니다.)

## 4. 통계 집계 서버 함수

`src/lib/stats.functions.ts`에 `getQuestionStats` 서버 함수를 추가합니다. 저장된 모든 질문을 불러와 키워드 사전으로 매칭한 뒤 다음을 계산해 반환합니다.

- **총 질문 횟수** — 전체 로그 수
- **인기 키워드 Top 10** — 키워드별 매칭 건수 + 비율(%)
- **최다 질문 키워드** — Top 1 키워드와 그 비율
- **일평균 질문 수** — 총 질문 수 ÷ 질문이 있었던 일수
- **최근 질문 목록** — 최신 5개 질문 원문 + 작성 시각(상대시간은 화면에서 표시)

## 5. 통계 페이지 실제 데이터 연결

`src/routes/stats.tsx`를 수정합니다.

- 라우트 `loader`에서 `getQuestionStats`를 호출해 캐시에 채우고, 컴포넌트는 `useSuspenseQuery`로 구독 (프로젝트의 TanStack Query 패턴 준수)
- 요약 카드(총 질문/최다 키워드/일평균), 막대그래프, 상세 표, 최근 질문 피드를 모두 실제 데이터로 교체
- **활성 학생 수 카드는 제거** (로그인이 없어 식별 불가)
- 데이터가 0건일 때를 위한 빈 상태(empty state) 메시지 추가
- 라우트에 `errorComponent` / `notFoundComponent` 추가 (프로젝트 규칙)

## 기술 메모

- 질문 기록·집계는 `supabaseAdmin`(서비스 역할)으로 서버 함수 핸들러 내부에서만 수행합니다.
- 키워드 매칭은 질문 원문을 저장해두고 **집계 시점에** 사전과 대조하는 방식이라, 사전을 수정하면 과거 질문에도 즉시 반영됩니다.
- 통계 페이지는 공개 라우트이므로 SSR 프리렌더 시에도 안전하게 동작하도록, 인증이 필요 없는 서버 함수로 구성합니다.
