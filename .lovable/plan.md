## 목표

키워드 통계가 사전에 미리 등록된 10개 주제(휴대폰·급식 등)만 잡는 문제를 해결합니다. **사전 없이 질문 원문에서 자주 나오는 단어를 자동으로 추출**해 통계로 만들어, "직업", "학교폭력"처럼 새로운 주제도 자동으로 잡히게 합니다.

## 1. 자동 키워드 추출 로직 (`src/lib/stats-keywords.ts` 교체)

기존 고정 사전(`KEYWORD_RULES`) 대신, 질문 텍스트에서 의미 있는 단어를 뽑는 함수로 바꿉니다.

추출 절차(빈도 기반, 추가 AI 비용 없음):
1. 질문을 한글/영문/숫자 단위 토큰으로 분리
2. 한글 토큰의 끝에 붙은 **조사 제거** (은/는/이/가/을/를/의/에/에서/으로/로/와/과/도/만/까지/부터/에게/한테/요/야 등) → "직업은", "직업의" 를 모두 "직업" 으로 합침
3. **불용어 제거**: 의문사·일반 동사 등 통계 의미 없는 단어 (어떤, 무엇, 언제, 어디, 어떻게, 왜, 종류, 경우, 그리고, 하면, 있어, 돼, 되, 가능, 수, 거, 게 등)
4. 2글자 이상 토큰만 사용

```text
"직업의 종류는 어떤 것들이 있어?"  →  ["직업"]
"직업은 몇개까지 가질 수 있어?"    →  ["직업"]
"휴대폰은 언제 쓸 수 있어?"        →  ["휴대폰"]
```

불용어/조사 목록은 한 곳에 상수로 모아 교육 연구용으로 쉽게 수정 가능하게 둡니다.

## 2. 집계 함수 수정 (`src/lib/stats.functions.ts`)

`matchKeywords` 대신 새 추출 함수를 호출하도록 변경합니다. 나머지 집계(총 질문 수, Top 10, 최다 키워드, 일평균, 최근 질문)는 그대로 동작합니다. 한 질문에서 같은 단어가 여러 번 나와도 1회로 셉니다.

## 3. 통계 페이지 하이드레이션 오류 수정 (`src/routes/stats.tsx`)

현재 "최근 질문"의 상대시간(`relativeTime`)이 서버 렌더와 클라이언트 렌더 시점 차이로 "5분 전 / 6분 전" 불일치를 일으켜 hydration 오류가 발생합니다. 마운트 완료(`isMounted`) 후에만 상대시간을 표시하고, 그 전에는 빈 값/고정 placeholder를 보여주도록 고쳐 오류를 없앱니다.

## 기술 메모

- 순수 문자열 처리라 서버 함수 안에서 안전하게 동작하고, 과거 질문에도 즉시 소급 적용됩니다(질문 원문 저장 방식 유지).
- 사전 매칭이 사라지므로 "비율(%)"은 추출된 전체 키워드 출현 수 기준으로 계산됩니다.
