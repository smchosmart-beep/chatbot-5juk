## 목표

키워드 통계에 "되는걸까", "정해" 같은 동사/형용사 활용형이 잡히는 문제를 없애고, **명사 위주로만** 키워드를 추출합니다. AI 없이 규칙(조사·어미 처리 + 동사 판별)만 강화합니다. 파일은 `src/lib/stats-keywords.ts` 한 곳만 수정합니다.

## 추출 알고리즘 개선

질문 한 건을 토큰으로 나눈 뒤, 각 토큰에 대해 순서대로 처리합니다.

1. **조사 제거** (기존 유지): "직업의" → "직업"
2. **동사/형용사 어미 제거하여 어간만 남기기**
   - "지각하면" → `하면` 제거 → "지각" ✓ (명사로 살림)
   - "되는걸까" → `는걸까` 제거 → "되" (1글자 → 다음 단계에서 탈락)
   - "정해" → `해` 제거 → "정" (1글자 → 탈락)
   - 처리할 어미 예: `하면/하는/하고/해서/해도/했어/한다/하지/되는걸까/되는/된다/돼서/될까/할까/나요/까요/는지/은지` 등
3. **명사 판별 필터(핵심 추가)**: 위 처리 뒤에도 토큰이 동사/형용사 활용 어미 잔재로 끝나면(예: `…다/…까/…지/…네/…죠/…해/…했/…돼/…됨/…자/…래/…봐` 등) 명사가 아니라고 보고 **버립니다.**
   - "되는걸까"·"정해"처럼 어간이 너무 짧아 탈락하는 경우 + 이 필터로 동사형이 확실히 제거됩니다.
4. **길이/불용어 필터** (기존 유지): 2글자 이상 + 불용어 목록 제외
5. 한 질문 안 중복 단어는 1회만 집계 (기존 유지)

## 동작 예시

```text
"직업의 종류는 어떤 것들이 있어?"      → [직업]
"내가 어떤 행동을 하면 학교폭력이 되는걸까?" → [행동, 학교폭력]   (되는걸까 제거)
"청소 당번은 어떻게 정해?"            → [청소, 당번]          (정해 제거)
"지각하면 어떻게 돼?"                 → [지각]
"수업 시간에 화장실 갈 수 있어?"       → [수업, 시간, 화장실]
```

## 기술 메모

- 조사 목록·동사 어미 목록·동사 종결 잔재 목록·불용어 목록을 모두 파일 상단 상수로 모아 교육 연구용으로 쉽게 수정할 수 있게 둡니다.
- 규칙 기반 특성상 "이해"·"오해"처럼 `해`로 끝나는 일부 명사가 함께 걸러질 수 있는데, 이런 단어는 필요 시 명사 예외(whitelist) 목록에 추가해 살릴 수 있도록 합니다.
- DB·집계 함수(`stats.functions.ts`)·통계 화면은 변경 없이 그대로 두고, 추출 로직만 교체하므로 과거 질문에도 즉시 소급 적용됩니다.
